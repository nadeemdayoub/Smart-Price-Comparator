# Impeccable Design Context

This file contains persistent design guidelines for the Smart Price Comparator project.

## Design Context

### Users
- **Primary**: Procurement teams at SMBs and enterprise organizations
- **Secondary**: Business owners, operations managers managing purchasing
- **Context**: Users make critical purchasing decisions daily, often under time pressure
- **Goal**: Upload supplier quotations, match products to catalog, compare prices across suppliers, identify best value

### Brand Personality
- **Voice**: Professional, direct, no-nonsense
- **Tone**: Authoritative yet approachable
- **Words**: Professional, Trustworthy, Efficient
- **Emotional Goal**: Instill confidence in purchasing decisions

### Aesthetic Direction
- **Visual Tone**: Clean, professional, enterprise-grade but not sterile
- **Avoid**: Generic AI aesthetics (gradients, blobs, "AI-powered" visual clichés)
- **Theme**: Light mode only (stone-based palette), no dark mode needed
- **Bilingual**: Supports English and Arabic (RTL)

### Color Strategy
- **Primary Accent**: Warm amber (amber-500) - conveys trust, optimism, financial success
- **CTA Buttons**: amber-500 bg with amber-600 on hover, shadow-amber-500/30
- **Active Navigation**: amber-50 bg with amber-500 left border accent
- **Semantic Colors**: emerald (savings), red (errors), blue (info), amber (warnings)
- **Warmth**: Subtle amber tints on backgrounds and accents

### Typography
- **Font Family (English)**: DIN Next LT Arabic - modern corporate sans-serif, excellent for business/procurement context
- **Font Family (Arabic)**: A Jannat LT - traditional Naskh-style Arabic for RTL content
- **Fallback English**: 'Segoe UI', Tahoma, Geneva, Verdana, sans-serif
- **Fallback Arabic**: 'Traditional Arabic', 'Arabic Typesetting', serif
- **Arabic**: Use `font-arabic` class or `dir="rtl"` on parent element
- **Mono**: Used for technical labels, references (`font-mono`)
- **Weights**: 400 (body), 500 (semibold), 600 (medium), 700 (bold/headings)
- **Type Scale**:
  - Display/Hero: text-5xl-7xl, font-bold, leading-[1.05], tracking-tight
  - Section Headers: text-heading-section (text-3xl lg:text-4xl, font-bold)
  - Card Titles: text-heading-card (text-xl, font-semibold)
  - Body: text-base, text-stone-600, leading-relaxed
  - Labels: text-label (text-xs, font-semibold, uppercase, tracking-wider)
- **Typography Utilities** (use via Tailwind classes):
  - `.text-label` - Uppercase labels with tracking
  - `.text-body` - Body text with relaxed leading
  - `.text-heading-display` - Hero headlines
  - `.text-heading-section` - Section headings
  - `.text-heading-card` - Card/subsection titles
- **Data tables**: Use `font-variant-numeric tabular-nums` for all price/number displays

### Design Principles
1. **Confidence through clarity** - Every element should help users trust their decisions. Use clear data visualization, obvious action paths, and visible confirmations.
2. **Efficiency is paramount** - Minimal clicks, immediate feedback, smart defaults. Respect users' time with fast, frictionless workflows.
3. **Professional restraint** - Subtle shadows, generous whitespace, restrained animations. Let data and functionality speak; don't decorate for decoration's sake.
4. **Consistency builds trust** - Same patterns everywhere: same button styles, same card layouts, same spacing rhythm. Predictability reduces cognitive load.
5. **Accessibility by default** - High contrast text, clear focus states, appropriate touch targets, motion respects reduced-motion preferences.

### Layout & Spacing
- **Page margins**: p-6 on mobile, p-8 on desktop
- **Card padding**: p-6 to p-8
- **Border radius**: rounded-xl (buttons), rounded-2xl (cards), rounded-3xl (large containers/modals)
- **Shadows**: shadow-sm (subtle), shadow-xl (floating elements), shadow-2xl (modals)
- **Grid gaps**: gap-4 to gap-8 depending on content density
- **Sidebar**: Fixed 256px (w-64) on desktop

### Component Patterns
- **Buttons**: 
  - Primary: bg-amber-500 text-white, hover:bg-amber-600, shadow-amber-500/30, rounded-xl
  - Secondary: bg-white border-stone-200, hover:bg-stone-50
  - Danger: bg-red-600 text-white, hover:bg-red-700
- **Cards**: bg-white rounded-2xl border border-stone-200 shadow-sm
- **Inputs**: bg-transparent border border-stone-200 rounded-xl, focus:ring-2 focus:ring-stone-900/5
- **Modals**: Fixed overlay bg-black/50 backdrop-blur-sm, centered rounded-3xl card
- **Tables**: bg-stone-50 header, divide-y divide-stone-100 rows, hover:bg-stone-50/50
- **Navigation**: 
  - Desktop: Fixed sidebar, active = bg-amber-50 text-amber-700 with amber-500 left border
  - Mobile: Fixed top bar, hamburger menu

### Animations
- **Library**: motion/react
- **Philosophy**: Subtle, purposeful. Entrance animations (fade + translate), hover lifts, modal scales
- **Reduced motion**: All animations respect `prefers-reduced-motion`
- **Duration**: 200-300ms for micro-interactions, 400-600ms for page transitions

### Icons
- **Library**: Lucide React
- **Size**: w-4/h-4 (nav/table), w-5/h-5 (buttons), w-6/h-6 (section icons), w-8/h-8 (feature cards)

### Motion Colors
- **Emerald**: Used for positive metrics, savings, success states
- **Red**: Used for price increases, errors, destructive actions
- **Amber**: Used for warnings, items needing attention
- **Blue**: Used for informational badges, links

### Accessibility
- **Focus states**: Visible ring on all interactive elements (ring-2 ring-stone-900/5)
- **Contrast**: All text meets WCAG AA standards
- **Touch targets**: Minimum 44x44px on mobile
- **Screen readers**: Proper ARIA labels on icons-only buttons
- **Error identification**: Color + icon + text for errors (never color alone)
